1. Бизнес процессы:
    1.1 Клиент подключается к серверу
    1.2 Клиент может запросить у сервера список меню
    1.3 Клиент с помощью команд формирует состав заказа
    1.4 Сервер получив от клиента список позиций формирует запрос на создание заказа
    1.5 Сервис создания заказа проверяет заказанные позиции на наличие остатков, цен и формирует заказ

2. Функциональные процессы
    1.1 Соединение клиента и сервера устанавливается с помощью технологии Socket для непрерывных передачи данных
    1.2 Клиент с помощью определенных команд может запросить список меню, сформировать состав заказа и отдать команду создания или отмены заказа
    1.3 Сервер получив от Клиента команду в текстовом виде, создает команду интерпертации
    1.4 Команда интерпретации определив полученную команду, запрашивает фукцию создания команды из IoC-контейнера либо сообщает о некорректности введенной команды
    1.5 С помощью полученной функции создается команда и в нее передаются требуемые параметры. Созданная команда выполняется в обработчике команд с непрерывным циклом
    1.6 После получения команды создания заказа, Сервер формирует запрос по технологии gRPC в Сервис создания заказа
    1.7 Сервис создания заказа, получив данные о заказе запускает процедуру проверки и создания заказа
    1.8 Процедура проверки основана на паттерне "Цепочка обязанностей". Последовательно проверяется наличие дубликата заказа, остатки и цены. В случае успеха формируется заказ и возвращается номер заказа
    1.9 Сервер, получив от Сервиса создания заказа номер заказа сообщает Клиенту номер его заказа
